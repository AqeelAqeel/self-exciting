### Card Actions

| Action | What It Does | Self-Exciting Effect |
|--------|--------------|---------------------|
| **Push** | Continue in this direction | Deepens commitment |
| **Fork** | Branch into new variant | Expands possibility |
| **Mutate** | Surprise me (same vibe) | Injects novelty |
| **Pin** | Save to workspace | Anchors preference |
| **Prune** | Kill this branch | Clarifies intent |

## Implicit Preference Learning

When the user drags, duplicates, or pins:

```
User Action          →    System Inference
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Select card          →    +0.1 weight on features
Duplicate card       →    +0.2 weight on features
Pin to workspace     →    +0.3 weight on features
Ignore card          →    -0.05 decay on features
Prune branch         →    -0.2 on orthogonal features
```

**No thumbs up. No ratings. Just action.**

This is selection pressure through behavior.

## Constraints (Non-Negotiable)

> Unbounded excitation = noise
> Bounded excitation = creation

| Constraint | Value | Why |
|------------|-------|-----|
| Max depth per direction | 5 | Prevents infinite rabbit holes |
| Max simultaneous generations | 3 | Maintains system responsiveness |
| Decay of old nodes | Visual fade | Focuses attention on frontier |
| Explicit commit action | Pin to workspace | Forces selection pressure |

## API Endpoints

| Endpoint | Method | Purpose |
|----------|--------|---------|
| `/api/salience/session` | POST | Create new session |
| `/api/salience/session/[id]` | GET | Get session state |
| `/api/salience/session/[id]/references` | POST | Upload references |
| `/api/salience/session/[id]/analyze` | POST | Extract salience + plan directions |
| `/api/salience/session/[id]/generate` | POST | Queue generation |
| `/api/salience/session/[id]/stream` | GET | SSE for real-time updates |
